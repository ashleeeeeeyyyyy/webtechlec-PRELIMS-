    HTTP

  • A protocol which use to communicate with the requrest of the client to the server and how the server will response.

  • Application layer communications protocol used to access resources on the WWW.

    • HTTP Stateless

    • Server do not keep information about client in-between request

HTTP provides support for other functionalities

  • Cache control
  • Content media type (MIME) specifications
  • Language and character set specifications
    • Other extended characters
  • Content/transfer coding
    • chunk details body,header
  • Content negotiation
  • Client Sever protocol negotiation
  • Persistent connectcions
  • Request pipelining/multiplexing
  • Authentication/authorization

HTTP Resources

Identifies using URIs

Parts:
  • Scheme(http or https)
  • Authority
    • user information or authentication credencials
    • host

      • can be domain name or IP address via DNS of the server

    • port number
  • Path to resource

    • may refer to static or dynamin

  • query

    • provided as key = value pars, with ampersand($)

    may be URL -encoded

  • Fragment Identifier

    • Example of this is the # symbol.

HTTP Request Message

https://www.w3.org/Protocols/rfc2616/rfc2616-sec5.html
  • Request Line
    • Method
    • Request Target
    • Protocol Versions
  • Message headers

    • HTTP 1.1 requires atleast the HOST request header to be provided

  • Empty Line

    • To show you are done with message headers

  • Message body a.k.a payload

HTTP Response

https://www.w3.org/Protocols/rfc2616/rfc2616-sec6.html
  • HTTP Response is similar with request message but instead of Request Line, it is called Static Line
  • Static line
    • Protocol Versions
    • Status Code
    • Reason Phase
  • Message headers
  • Empty Line
  • Message body a.k.a payload

HTTP Request Methods

Get

  • Most commonly used method.

  • Get the message body.

    • Head

  • Identical to GET, except that the server does not send message body.

  • Only get the header information, used to retrieve metadata.

    • Post

  • Same us GET method.

  • Used in submitting HTML form data

  • Data that submit is now in the body

    • PUT

  • Create or replace.

    • Delete

  • Remove the association between the target.

    • Options

  • Options available for the target resource.

  • Inquiring the available methods that can be use.

    • Trace

  • Request a remote, application-level loop-back of the request message

  • Logically used for testing.

    • Connect

  • Request the established of a tunnel.

    • Methods Properties

    Safe Methods

    Client does not expect change (read only).

    Methods:
    • Get
    • Head
    • Options
    • Trace

    Indepotent Methods

    Intended effect on the server with the method is the same as the effect of a single request.

    Methods:
    • Get
    • Head
    • Option
    • Trace
    • Put
    • Delete

    Cacheable Methods

    Indicates the response to a method is doved to be store for future reuse.

    Methods
    • Get
    • Head
    • Post

    HTTP Status Code

    3 digit numbers

    1. Informational (1XX)

      • Server is thinking throught the request

      • List of 100:
        • 100 Continue
        • 101 Switching Protocols
    2. Success (2XX)

      • Client's request was successfully received.

      • List of 200:
        • 200 Ok
        • 201 Created
        • 202 Accepted
        • 203 Non-Authoritative Information
        • 204 No Conent
        • 205 Reset Content
        • 206 Partial Content
    3. Redirection (3XX)

      • Further action needs to be taken by the

      • List of 300:
        • 300 Multiple Choices
        • 301 Moved Permanently
        • 302 Found
        • 303 See Other
        • 304 Not Modified
        • 305 Use Proxy
        • 307 Temporary Redirect
    4. Client Error (4XX)

      • The client made a mistake which lead to an error.

        • 400 Bad Request
        • 401 Unauthorized
        • 402 Payment Required
        • 403 Forbidden
        • 404 Not Found
        • 405 Method Not Allowed
        • 406 Not Acceptable
        • 407 Proxy Authentication Required
        • 408 Request Time-out
        • 409 Conflict
        • 410 Gone
        • 411 Length Required
        • 412 Precondition Failed
        • 413 Request Entity Too Large
        • 414 Request-URI Too Long
        • 415 Unsupported Media Type
        • 416 Request Range Not Satisfiable
        • 417 Expectation Failed
    5. Server Error (5XX)

      • The server failed to complete the request

      • List of 500:
        • 500 Internal Server Error
        • 501 Not Implemented
        • 502 Bad Gateway
        • 503 Service Unavailable
        • 504 Gateway Time-out
        • 505 HTTP Version Not Supported

    • HTTP can be extended by defining new request, methods, message headers, and/or status codes

    • example:

      WebDAV(RFC 4918)

      • Request Methods

        • PROPFIND, PROPPATCH, MKCOL, COPY, MOVE, LOCK, UNLOCK

      • Message Headers

        • DAV, Depth, Destination, Overwrite, Timeout

      • Status Codes

        • Multi-Status (207), Locked (423)
