HTTP

  • A protocol which use to communicate with the requrest of the client to the server and how the server will response.

  • Application layer communications protocol used to access resources on the WWW.

    • HTTP Stateless

    HTTP Resources

    Identifies using URIs

    Parts:
    • Scheme(http or https)
    • Authority
      • user information or authentication credencials
      • host

        • can be domain name or IP address via DNS of the server

      • port number
    • Path to resource

      • may refer to static or dynamin

    • query

      • provided as key = value pars, with ampersand($)

      may be URL -encoded

    • Fragment Identifier

      • Example of this is the # symbol.

    HTTP Request Message

    • Request Line
    • Message headers
    • Empty Line
    • Message body a.k.a payload

    HTTP Response

    • Static line
    • Message headers
    • Empty Line
    • Message body a.k.a payload

    HTTP Request Methods

    Get

  • Most commonly used method.

  • Get the message body.

    • Head

  • Identical to GET, except that the server does not send message body.

  • Only get the header information, used to retrieve metadata.

    • Post

  • Same us GET method.

  • Used in submitting HTML form data

  • Data that submit is now in the body

    • PUT

  • Create or replace.

    • Delete

  • Remove the association between the target.

    • Options

  • Options available for the target resource.

  • Inquiring the available methods that can be use.

    • Trace

  • Request a remote, application-level loop-back of the request message

  • Logically used for testing.

    • Connect

  • Request the established of a tunnel.

    • Methods Properties

    Safe Methods

    Client does not expect change (read only).

    Methods:
    • Get
    • Head
    • Options
    • Trace

    Indepotent Methods

    Intended effect on the server with the method is the same as the effect of a single request.

    Methods:
    • Get
    • Head
    • Option
    • Trace
    • Put
    • Delete

    Cacheable Methods

    Indicates the response to a method is doved to be store for future reuse.

    Methods
    • Get
    • Head
    • Post

    HTTP Status Code

    3 digit numbers

    1. Informational (1XX)
    2. Success (2XX)
    3. Redirection (3XX)
    4. Client Error (4XX)
    5. Server Error (5XX)
